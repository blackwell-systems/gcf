# Independent Review: Gemini

**Date:** 2026-06-10
**Model:** Gemini (fresh session, no context)
**Specs reviewed:** RFC 8259 (JSON), TOON v3.3, GCF v2.0
**Verdict:** Picked GCF for AI agent tool response replacement

## Rankings

| Category | Winner |
|----------|--------|
| Nested/mixed data | **GCF** |
| JSON round-trip fidelity | **GCF** |
| Graph at scale (500+) | **GCF** |
| Spec rigor | TOON |
| Agent replacement pick | **GCF** |

## Key quotes

> "For AI agent tool responses, GCF is the clear choice."

> "GCF handles nested data in its generic profile using a 'Tabular with Attachments' model. This maintains the token efficiency of a table without forcing the entire structure to flatten."

> "GCF is the only format that explicitly differentiates between a null field and a missing field."

> "GCF is vastly superior here." (on graph relationships at 500+ records)

> "AI interactions are frequently stateful and involve navigating relationships. GCF's support for Session Statefulness and Delta Encoding allows an agent to see only what changed between turns, which is a critical feature for long-running agentic workflows."

## Full review

### 1. Strongest and Weakest Design Decisions

| Format | Strongest | Weakest |
|--------|-----------|---------|
| JSON | Ubiquity. Its rigid, simple data model is the "lingua franca" of APIs. | Token Tax. Repetition of keys and heavy use of structural delimiters wastes context window. |
| TOON | Tabular Syntax. Defining the schema once in the header is a major improvement for uniform data. | List-Item Complexity. The "Objects as List Items" rules (§10) are complex and context-sensitive. |
| GCF | Graph Specialization. The dedicated graph profile with local @IDs and distance-based grouping is highly optimized for AI agent code exploration. | Rigid Indentation. Exactly two-space indentation (§4.1) and complex attachment syntax increase implementation difficulty. |

### 2. Handling Nested and Mixed-Type Data

Winner: GCF. The "Tabular with Attachments" model (Section 7.4.4) maintains token efficiency without forcing structure to flatten. TOON's approach falls back to an "expanded list" that loses schema-definition benefits.

### 3. JSON Round-Trip Fidelity

Winner: GCF. The only format that explicitly differentiates null from missing (- vs ~). TOON's tabular mode decodes empty tokens to empty strings, causing data loss.

### 4. Graph Relationships at Scale

Winner: GCF. Vastly superior. Local IDs prevent repeating 100+ character names. Implicit distance grouping via section headers. Compact edge directionality syntax.

### 5. Spec Rigor

Winner: TOON. Cited canonical number formatting (§2), key folding and path expansion (§13.4), strict mode (§14).

Note: GCF has canonical number formatting (Section 2.3.1) and strict mode (Section 16.5, 30 error conditions) but the reviewer did not identify them, suggesting they were not prominent enough in the spec. This has since been addressed with explicit section headings.

### 6. Missing or Underspecified

- TOON: missing null vs missing distinction. Non-uniform data in tabular rows forces verbose expanded format.
- GCF: kind abbreviation extension mechanism is vague. Indentation rules intolerant of variation.

### 7. The Replacement Pick

GCF. "GCF's support for Session Statefulness and Delta Encoding allows an agent to see only what changed between turns, which is a critical feature for long-running agentic workflows. While TOON is a great 'prettier CSV' for logs, GCF's Graph Profile solves the specific 'token tax' problem that currently limits the scale of code-graph and knowledge-graph interactions in AI pipelines."
